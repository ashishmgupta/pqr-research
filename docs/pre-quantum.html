Pre-Quantum Cryptography

Before I could understand what post-quantum cryptography changes, I needed to be really clear on what it's replacing. I thought I knew how TLS worked. Turns out I had a few gaps. This section is what I actually learned — starting from the handshake, through RSA, through Shor's algorithm, and ending at the attack that doesn't need a quantum computer to start.

How a TLS Handshake Works Today

The key thing that clicked for me: the TLS handshake is solving two completely separate problems at once. First, key exchange — how do two strangers on the internet agree on a shared secret without anyone intercepting it? And second, data encryption — once they have that secret, how do they use it to protect everything they say to each other? Post-quantum cryptography only touches the first one. I'll come back to why that matters.

🌐 Browser / Client
Chrome, Firefox, Edge…
🖥️ Web Server
Cloudflare · F5 · Origin
1
TCP Handshake — before TLS even starts
SYN → SYN-ACK → ACK. A reliable transport connection is established. No encryption yet.
2
Client Hello → Server
Browser sends: supported TLS versions, list of cipher suites it supports (e.g. TLS_AES_256_GCM_SHA384), and its supported key agreement groups (e.g. x25519, P-256). In TLS 1.3 it also sends a speculative key share to avoid a round-trip.
3
Server Hello + Certificate ← Server
Server picks a cipher suite, responds with its public key certificate (RSA or ECDSA signed by a trusted CA). The certificate proves the server's identity.
4
Key Exchange — ⚠️ Quantum-Vulnerable Step
RSA mode: Browser generates a random secret, encrypts it with the server's public key, and sends the ciphertext. Only the server can decrypt it with its private key.

ECDH mode (~70–80% of modern traffic): Both sides contribute key material via Diffie-Hellman math. The shared secret is computed, never transmitted. Either way — Shor's Algorithm on a quantum computer breaks both by recovering the server's private key.
5
Derive Session Keys
Both sides now hold the same shared secret. They independently run it through a key derivation function (HKDF) to produce symmetric encryption keys. No key material crosses the network in this step.
6
AES-256-GCM — ✅ Quantum-Safe
All HTTP traffic — GET requests, HTML, login forms, API calls — is now encrypted with AES-256 using the derived session key. AES-256 is not vulnerable to quantum computers. Grover's algorithm only halves the effective key length (256 → 128 bits), which is still secure.
The thing that surprised me: Step 4 and Step 6 are completely separate problems, and quantum computers only break Step 4. Your actual data in Step 6 is encrypted with AES-256 and that's fine. Post-quantum cryptography replaces Step 4 only — the rest of TLS is untouched. It's a targeted fix, not a rebuild from scratch.

RSA: Security by Factoring Difficulty

Here's the bit that made RSA click for me. The security isn't magic — it's a maths problem that's easy in one direction and brutally hard in reverse. Pick two large prime numbers, multiply them together, and you get n. That's fast. Trivially fast. But if I hand you n and ask you to find the two primes I started with? That's the problem that's kept RSA secure for decades. The table below shows how quickly that difficulty scales.

🔑 RSA Key Generation

Pick two large primes p and q.
Compute n = p × q — this is the public modulus.
Publish n as part of the public key.
Keep p and q secret forever.

p and q are each ~300–600 digits.

The Quantum Threat: Shor's Algorithm

So if factoring large numbers is the problem, what does a quantum computer do differently? In 1994, a mathematician named Peter Shor figured out that quantum computers could solve the factoring problem exponentially faster — not by trying harder, but by exploiting three properties of quantum mechanics that have no equivalent in traditional computing. I'm not a physicist, but here's how I think about each one.

🔵 Superposition

A qubit exists as 0 and 1 simultaneously until measured. Shor's algorithm evaluates a function at all possible inputs at once, rather than one at a time.

🔗 Entanglement

Qubits become linked — measuring one instantly determines entangled partners. Shor's algorithm uses this to correlate results across the superposition.

〰️ Interference

Quantum algorithms amplify correct answers and cancel wrong ones through phase manipulation. The period — and thus the factors — emerges from this process.

What actually breaks and what doesn't: RSA and ECDH both rely on mathematical problems that Shor's algorithm dismantles — integer factorization and discrete logarithm respectively. AES-256 is different. The best quantum attack against it (Grover's algorithm) only halves the effective key length — 256 bits becomes 128 bits, still secure. So your encrypted data isn't the concern. The key exchange is.

⚠️ Where we actually are in 2025

The largest number ever factored using Shor's Algorithm on real quantum hardware is ~21. To break RSA-2048, you'd need roughly 4,000 logical qubits — which translates to millions of physical qubits once you account for error correction. Most estimates put a cryptographically capable quantum computer 15–20+ years away.

So why are we talking about this now? Read on.

Harvest Now, Decrypt Later

This is the part that shifted my thinking from "interesting future problem" to "we should be doing something about this now." You don't need a quantum computer to start the attack. You just need storage and patience — and nation-state actors have both.

1
Capture
Adversary archives encrypted TLS sessions from internet taps, ISP access, or BGP hijacking. This is happening today.
2
Store
The ciphertext is kept. It contains the RSA or ECDH key exchange — all they need to later derive the session key.
3
Wait
Adversary waits until a cryptographically capable quantum computer exists. Estimated: 2035–2040+.
4
Decrypt
Run Shor's Algorithm. Recover the session key. Decrypt the archived HTTPS traffic — login sessions, documents, financial data — retroactively.
Ask yourself: what data are you protecting today that still needs to be secret in 2040? Health records. M&A conversations. Government communications. Trade secrets. If the answer is anything, the migration clock is already running — because the capture is already happening. The timeline to migrate to post-quantum is measured in years, not weeks.
