✓ Verified on loginv2.lpl.com — April 2026

The command below is how I confirmed ML-KEM is actually being negotiated in production, not just theoretically supported. The key line to look for is Negotiated TLS1.3 group: X25519MLKEM768.

Requires OpenSSL 3.5+ — see the OpenSSL setup guide if you hit errors. Running in Git Bash on Windows or building from source on WSL are both covered. 

LD_LIBRARY_PATH=/usr/local/openssl35/lib64 \
  /usr/local/openssl35/bin/openssl s_client \
  -connect loginv2.lpl.com:443 \
  -tls1_3 -groups "X25519MLKEM768:x25519" \
  -CAfile /etc/ssl/certs/ca-certificates.crt \
  -brief

Negotiated TLS1.3 group: X25519MLKEM768  ← this is what you want to see
